# Contributing to create-mern-app

Thank you for your interest in contributing! This project is open source and we welcome contributions of all kinds.

## How to contribute

### Reporting bugs

1. Check if the bug has already been [reported](https://github.com/rohitnirban/create-mern-app/issues).
2. If not, open a new issue with a clear title and description.
3. Include steps to reproduce, your environment (OS, Node version), and any relevant output.

### Suggesting features

1. Open an issue describing the feature and its use case.
2. Wait for discussion before implementing large changes.

### Pull requests

1. **Fork** the repository and clone your fork.
2. Create a **branch**: `git checkout -b fix/some-bug` or `git checkout -b feat/some-feature`.
3. Make your changes. Keep commits focused and messages clear.
4. Run **tests**: `npm test`
5. Run **build**: `npm run build`
6. **Push** to your fork and open a pull request against `main`.
7. Describe your changes and link any related issues.

### Code style

- TypeScript is preferred for new code.
- Use existing patterns in the codebase.
- Keep the test suite passing.

## Development setup

```bash
git clone https://github.com/rohitnirban/create-mern-app.git
cd create-mern-app
npm install
npm run build
npm test
```

Open a [discussion](https://github.com/rohitnirban/create-mern-app/discussions) or an issue.
